## 1. How do you fetch data into AWS from a mobile application?

I expose a secure API endpoint using API Gateway and process data using Lambda or EC2.

### Architecture Flow:
Mobile App → API Gateway → Lambda/EC2 → S3 → Aurora → Tableau

### Steps:
- Create REST API using API Gateway
- Secure with Cognito / JWT / API Keys
- Process request in Lambda
- Store raw JSON in S3 (for audit & replay)
- Store structured data in Aurora (RDS)
- Enable CloudWatch logging and monitoring

## 2. How do you provide access to users in AWS?

Use IAM with least-privilege access control.

### Internal Users:
- Create IAM Groups (Admin, ReadOnly, DevOps, DataTeam)
- Attach policies to groups
- Add users to groups

### Application Access:
- Use IAM Roles
- Attach roles to EC2, Lambda, EKS

### Database Access:
- Create DB users
- Provide read-only access for reporting tools

### Audit:
- Enable CloudTrail
- Enable S3 access logs
- Enable RDS audit logs

## 3. How to provide access using Terraform?

Use Terraform to automate IAM configuration.

### Create IAM Policy
resource "aws_iam_policy" "s3_read" {
  name = "S3ReadPolicy"
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Action = ["s3:GetObject", "s3:ListBucket"]
      Resource = "*"
    }]
  })
}

### Create IAM Group
resource "aws_iam_group" "data_team" {
  name = "data-team"
}

### Attach Policy
resource "aws_iam_group_policy_attachment" "attach" {
  group      = aws_iam_group.data_team.name
  policy_arn = aws_iam_policy.s3_read.arn
}

### Create IAM Role for EC2
resource "aws_iam_role" "ec2_role" {
  name = "ec2-role"
  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Principal = { Service = "ec2.amazonaws.com" }
      Action = "sts:AssumeRole"
    }]
  })
}

## 4. Upgrade EC2 from L3 family to M5 family using Terraform

Modify instance_type in Terraform.

Before:
instance_type = "l3.large"

After:
instance_type = "m5.large"

Apply:
terraform plan
terraform apply

Production-safe approach:
- Update Launch Template
- Trigger Auto Scaling rolling refresh

## 5. SQL Query – Monthly Attendance Report (Present & Absent Count)

Requirement:
Count total present and absent students for a particular class in a specific month.

SELECT 
    s.class_no,
    SUM(CASE WHEN a.attendance_status = 'P' THEN 1 ELSE 0 END) AS total_present,
    SUM(CASE WHEN a.attendance_status = 'A' THEN 1 ELSE 0 END) AS total_absent
FROM attendance_audit a
JOIN student_info s 
    ON a.student_id = s.student_id
WHERE s.class_no = 10
  AND a.attendance_date BETWEEN '2025-01-01' AND '2025-01-31'
GROUP BY s.class_no;

Best practice:
Use date range filtering instead of MONTH() for better performance.

## 6. What is AWS Bedrock?

Amazon Bedrock is a fully managed generative AI service that provides access to foundation models via API.

### Features:
- Serverless
- No infrastructure management
- IAM integrated security
- Supports RAG and fine-tuning
- Access to models like Anthropic, Meta, Amazon Titan

### Use Cases:
- Chatbots
- AI assistants
- Text generation
- Code generation
- Knowledge base Q&A
